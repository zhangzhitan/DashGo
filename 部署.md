这两个 python -c 命令主要用于初始化 Plotly-DashGo 应用的数据库环境，确保应用启动时有必要的表结构和初始管理员账户。

具体来说，它们分别执行了 src/database/sql_db/conn.py 文件中定义的两个关键函数：

1. create_rds_table() —— 创建数据库表结构
这个命令的作用是构建“骨架”。

它会连接数据库，并检查所有必要的表是否存在。

如果表不存在，它会自动创建。

它创建的表涵盖了系统的核心模块，包括：

用户与权限：如 SysUser（用户表）、SysRole（角色表）、SysUserRole（关联表）。

任务管理：如 SysApschedulerResults（任务结果）、SysListenTask（监听任务）。

系统功能：如 SysAnnouncement（系统公告）、OAuth2 相关认证表。

2. init_rds_data() —— 写入初始数据
这个命令的作用是注入“灵魂”，即预置种子数据。

它确保系统不是空的，让你可以立即登录使用。

具体操作包括：

创建角色：新增一个名为 admin 的角色（备注为“超级管理员角色”）。

创建管理员：新增一个用户名为 admin 的超级管理员用户。
 
设置默认密码：将该用户的密码初始化为 admin123（存储时会进行 SHA256 加密）。

赋予权限：将 admin 用户与 admin 角色绑定。

总结： 执行这两个命令后，你的数据库就准备就绪了，你可以直接使用账号 admin 和密码 admin123 登录系统。


针对你在 Plotly-DashGo 框架下新增“数据盘点”页面的需求，你需要按照框架现有的 MVC 模式（Model-View-Callback）进行开发。

这个需求的核心点在于：

后端分页：利用 SQL 的 LIMIT/OFFSET 实现高性能查询。

Excel 解析：使用 pandas 读取上传内容。

细粒度权限控制：通过代码逻辑判断是否渲染“导入”按钮。
